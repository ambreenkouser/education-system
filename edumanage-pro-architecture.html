2Communication Strategy

Synchronous вҖ” REST via Feign Client

Used when an immediate response is required between services.

CallerCalleePurpose
api-gatewayauth-serviceToken validation on every request
api-gatewayuser-serviceProfile enrichment
student-servicecourse-serviceEnrollment eligibility check
grade-servicestudent-serviceStudent lookup before grade assignment
fee-servicestudent-serviceStudent validation before invoice
timetable-servicecourse-serviceCourse detail fetch for scheduling

Asynchronous вҖ” Apache Kafka

Used for decoupled, event-driven flows. Cluster: 3 brokers, replication factor 3, 6 partitions per topic.

student.enrolled
Producer: student-service
Consumers: fee-service, notification-service, timetable-service
attendance.marked
Producer: attendance-service
Consumers: notification-service, report-service

Consumers: notification-service, report-service

Producer: fee-service
Consumers: notification-service, report-service
user.created
Producer: auth-service
Consumers: user-service, notification-service

/api/attendance/**attendance-serviceROLE_TEACHER, ROLE_ADMIN

Gateway Filter Chain

AuthenticationFilter    вҶ’ validates JWT, injects X-User-Id & X-User-Role headers
RateLimitFilter         вҶ’ Redis token bucket (per IP + per user)
CircuitBreakerFilter    вҶ’ Resilience4j per route
RequestLoggingFilter    вҶ’ X-Correlation-ID propagation across all services

Redis Usage in Gateway

PurposeTTL
Rate limiting counters1 minute sliding window
JWT blacklist (logout / token revoke)Token expiry duration
Downstream service response cache30 seconds (public routes only)
4Database Strategy

Pattern: Database-per-Service вҖ” each service owns its schema, no shared tables.

ServiceDatabaseRationale
auth-servicePostgreSQLRelational вҖ” users, roles, refresh tokens
user-servicePostgreSQLRelational вҖ” profiles, addresses, contacts
student-servicePostgreSQLRelational вҖ” enrollment, parent relations
course-servicePostgreSQLRelational вҖ” curriculum, subject hierarchy
timetable-servicePostgreSQLRelational вҖ” schedule constraints, rooms
attendance-servicePostgreSQL + RedisPG for persistence; Redis caches today's live data
grade-servicePostgreSQLRelational вҖ” exam records, GPA history
fee-servicePostgreSQLACID critical вҖ” financial transactions
notification-serviceRedisEphemeral вҖ” queued/sent notification state
report-servicePostgreSQL (read replica)Aggregation-heavy; separate from OLTP writes

Design Patterns Applied

PatternApplied ToPurpose
Saga (Choreography)student-service вҶ’ fee-service вҶ’ notification-serviceMulti-service enrollment without distributed transactions
Outbox Patternfee-service, grade-serviceReliable Kafka event publishing вҖ” no dual-write risk
CQRSreport-serviceWrite via Kafka events; read from materialized views
Flyway MigrationsAll servicesVersion-controlled schema evolution
HikariCP PoolingAll servicesTuned max-pool-size per service load profile
